# 🎨 现代前端设计规范 SOP (Frontend Design SOP)

本规范基于顶尖设计品味与前沿全栈工程实践，旨在彻底铲除“上世纪后台管理系统”的粗糙风格，为企业级应用注入现代化、呼吸感与高级感。

## 一、核心视觉法则 (Visual Foundations)

### 1. 颜色与对比度 (Colors & Contrast)
- **摒弃高饱和度单色**：禁止使用纯大红(`#FF0000`)、纯蓝(`#0000FF`)。使用基于 HSB/HSL 空间调和的色板（如 Tailwind 的 Slate、Zinc 等高级纯净底色），或者高质感莫兰迪色系。
- **打破“非黑即白”**：大面积背景色禁止使用死白(`#FFFFFF`)或死黑(`#000000`)。背景色建议使用 `#F9FAFB` (灰白) 或 `#F3F4F6` 进行柔和托底，卡片层使用 `#FFFFFF` 形成层次；文字颜色使用 `#111827` (主标题)、`#4B5563` (次文本) 代替高刺鼻的纯黑。
- **无障碍访问特性 (a11y)**：文本与其底层背景的对比度需满足 WCAG 2.1 的 AA 标准（至少 4.5:1）。

### 2. 呼吸感与留白 (Spacing)
- 严格遵循 **8点网格系统 (8pt Grid System)**。外边距(Margin)和内边距(Padding)的增量应优先基于 4, 8, 12, 16, 24, 32, 48 等阶梯梯队。
- 绝不允许毫无间隙的 UI 元素堆砌，适当的留白（Negative Space）是提升界面高级感、降低信息压迫感最具性价比的手段。

### 3. 圆角与阴影 (Border Radius & Elevation)
- **和谐的圆角**：现代 UI 抛弃了直角矩形的生硬与冰冷感。主卡片使用 `12px~16px` 左右圆角（如 Tailwind 的 `rounded-xl` / `rounded-2xl`），内部嵌套元素使用 `6px~8px`，大画幅面板放开至 `16px~24px`。强烈要求保持容器内外层圆角率收缩的**视觉同心圆**效果。
- **弥散阴影**：彻底抛弃死板的短促黑阴影（如 `box-shadow: 0 2px 4px #000`）。推荐使用带有较大模糊半径与底层发散特性的“低重力弥散阴影”（如 `0 10px 15px -3px rgba(0, 0, 0, 0.05)`），营造干净剔透的物理悬浮空间布局。

## 二、Vue3 + Element Plus 的现修改造防坑指南

传统后台组件库（Element Plus, AntD）不可避免地带有早年的重度边框与僵硬特性。我们需要运用 CSS 变量对其默认 Token 进行局部覆盖，完成“去油化”。

1. **主题降噪**：重写全局边框主色 `--el-border-color-light` / `--el-border-color-lighter`，降低边框饱和度使其变得更柔和透亮。
2. **卡片剔透化**：修改 `.el-card` 的默认样式，干掉生硬的实体 Border（`border: none;`），统一切换为前文所述的弥散浅阴影与更平滑的 Border-radius。
3. **按钮灵动感**：给核心 `.el-button` 添加硬件加速动画 `transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);`，鼠标悬浮时产生轻微的微缩放效果（`transform: translateY(-2px);`）并释放一圈底部色影。

## 三、高级动效与交互 (Micro-interactions)
- **禁止木僵反馈**：任何具有交互闭环的按钮、表格行(`tr`)、数据卡片，被 Hover 时必须给予立即的色彩叠加垫底和 `cursor: pointer` 指示变化。
- **平滑渐进形态**：使用 `transition` 控制所有元素出现与隐藏的突兀形态（如弹窗 `el-dialog` 采用阻尼动画缩放渐显、路由切换提供淡入淡出 `fade-transform` 过渡），动作时长死掐在最舒适的 `200ms~300ms` 区间内。

## 四、响应式与设备适配 (Responsiveness)
- **拒绝流式崩坏**：使用 Flex/Grid 而非落后的 Float 堆叠布局。大显示器环境应设好 `max-width` 的天花板强制居中布局（如 `1440px` 或 `1600px`），防止文本与表单在大带鱼屏下极度稀疏、丑陋拉长。
- **移动端收束兜底**：若检测到非 PC 分辨率（`< 1024px`），须确保左侧导航支持收束。所有的横向无限延伸的数据大表，必须通过父容器的 `overflow-x: auto` 形成内部滚动，死守页面主体的宽度防御机制。
