3. 社保扣除自动化计算
社保（五险一金）不可手动填入。系统需根据该员工所属部门的“基本工资”（即社保缴费基数），乘以固定的个人缴纳比例，自动得出本月应扣社保金额
步骤 1：设定五险一金个人缴纳总比例
    定义常量 个人社保总比例 = 0.175  // (养老8%，医疗2%，失业0.5%，公积金7%即 17.5%)

步骤 2：执行乘法计算
    定义 应扣社保金额 = 基本工资 * 个人社保总比例
将【应扣社保金额】自动填充至核算页面，并设为不可修改

4. 基于“入职时间”的个税累计预扣
要实现的功能：
系统要体现国家现行的“跨月累计预扣法”。不考虑专项附加扣除。系统需要根据员工的“入职时间”推算本年度已入职的月份数（用于计算免税额度），并结合数据库中历史月份的薪资数据，算出本月实际应缴个税。 默认入职前，该年总工资为0元
逻辑实现说明（伪代码）：
【输入数据】: 员工入职时间、本月应发总额、本月应扣社保
【预期输出】: 本月实际应扣个税

步骤 1：查询该员工本年度的历史累计数据（通过单条 SQL SUM 语句实现）
    查询数据库：该员工从本年 1 月到上个月的【累计历史应发工资】、【累计历史已扣社保】、【累计历史已交个税】
    （注：如果是首月发工资，这三个查出来的值均为 0）

步骤 2：计算累计免税起征点
    根据 (当前计算月份) 与 (员工入职时间) 算出本年度“已入职月数 N”
    定义 累计免税额 = N * 5000

步骤 3：计算目前的“累计应纳税所得额”
    定义 本年累计总收入 = 累计历史应发工资 + 本月应发总工资
    定义 本年累计总扣社保 = 累计历史已扣社保 + 本月应扣社保
    定义 累计应纳税所得额 = 本年累计总收入 - 本年累计总扣社保 - 累计免税额

步骤 4：套用国家阶梯税率表，计算本年总共该交的税
    定义 本年应交个税总额 = 0
    如果 (累计应纳税所得额 > 0) 则：
        如果 (累计应纳税所得额 <= 36000) 则：
            本年应交个税总额 = 累计应纳税所得额 * 3%
        否则 如果 (累计应纳税所得额 <= 144000) 则：
            本年应交个税总额 = (累计应纳税所得额 * 10%) - 2520 
        否则 如果 (累计应纳税所得额 <= 300000) 则：
            本年应交个税总额 = (累计应纳税所得额 * 20%) - 16920
        // ...更高档次省略...

步骤 5：得出本月最终应扣个税
    定义 本月实际应扣个税 = 本年应交个税总额 - 累计历史已交个税
    如果 (本月实际应扣个税 < 0) 则：
        本月实际应扣个税 = 0
